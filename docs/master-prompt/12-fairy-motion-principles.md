# Pairy - 페어리/마법 모션 원칙 (Fairy Motion Principles)

## 개요

이 문서는 Pairy 플랫폼의 **모션 디자인 철학과 물리 법칙**을 정의합니다.
현실 세계의 물리가 아닌, **마법 세계의 물리학**을 기반으로 모든 인터랙션을 설계합니다.

> **핵심 정체성**: 페어리, 마법, 빛, 반짝임
> **모든 모션은 이 정체성을 표현해야 합니다.**

---

## Part 1: 마법 물리학 (Fairy Physics)

### 1.1 중력의 재정의

현실 세계에서 물체는 아래로 떨어집니다. 마법 세계에서는 다릅니다.

| 물리 속성 | 현실 세계 | 마법 세계 |
|----------|----------|----------|
| **중력 방향** | 항상 아래 | 상황에 따라 위/옆/사방 |
| **낙하 속도** | 가속도 9.8m/s² | 천천히, 떠다니듯 |
| **질량 영향** | 무거우면 빨리 떨어짐 | 빛나는 것은 떠오름 |
| **저항** | 공기 저항 | 마법 저항 (더 부드러움) |

```
마법 중력 공식:
- 빛을 발하는 요소 → 위로 떠오르는 경향
- 반짝이는 요소 → 사방으로 흩어지는 경향
- 활성화된 요소 → 중심에서 바깥으로 퍼지는 경향
```

### 1.2 마법 가속도 곡선

현실의 가속/감속과 다른 마법만의 속도 변화:

```
┌─────────────────────────────────────────────────────────┐
│  현실 물리 (스프링)          마법 물리 (글로우)          │
│                                                         │
│  속도                        속도                        │
│   ▲    ╱╲                     ▲      ___                │
│   │   ╱  ╲  ╱╲                │    ╱     ╲              │
│   │  ╱    ╲╱  ╲               │  ╱         ╲            │
│   │ ╱          ╲              │╱             ╲___       │
│   └─────────────► 시간        └─────────────────► 시간   │
│                                                         │
│  급격한 변화 + 반동           부드러운 시작 + 여운        │
└─────────────────────────────────────────────────────────┘
```

#### 마법 가속도 원칙
1. **부드러운 시작 (Soft Begin)**: 갑자기 시작하지 않음, 서서히 깨어남
2. **우아한 정점 (Graceful Peak)**: 급격한 피크 없음, 부드럽게 최고점 도달
3. **여운 있는 끝 (Lingering End)**: 갑자기 멈추지 않음, 잔향처럼 사라짐
4. **반동 없음 (No Bounce)**: 스프링처럼 튀지 않음, 물결처럼 흐름

### 1.3 빛의 물리학

마법 세계에서 빛은 물질처럼 행동합니다:

```
빛의 행동 양식:

1. 발산 (Emanation)
   중심에서 바깥으로 퍼져나감
   ○ → ◐ → ● → ○●○ → ○ ● ○

2. 맥동 (Pulsation)
   숨쉬듯 밝아졌다 어두워짐
   ◐ → ● → ◐ → ○ → ◐ → ●

3. 잔상 (Afterglow)
   빛이 사라진 후에도 흔적이 남음
   ●●● → ●●○ → ●○○ → ○○○ → ·○○

4. 굴절 (Refraction)
   하나의 빛이 여러 색으로 분리됨
   ● → 🔴🟡🟢🔵🟣
```

### 1.4 파티클 물리학

요정 가루와 별가루의 움직임 법칙:

```
파티클 생명주기:

  탄생        성장         정점         소멸
   ·    →    ●    →     ✦    →      ·

  0.0s      0.2s       0.4s       0.8s

탄생: 중심에서 작게 나타남 (scale 0 → 0.5)
성장: 이동하며 커짐 (scale 0.5 → 1.0)
정점: 최대 밝기, 최대 크기
소멸: 흐려지며 작아짐 (opacity 1 → 0, scale 1 → 0.3)
```

#### 파티클 분산 패턴

```
방사형 (Radial)           나선형 (Spiral)          부유형 (Float)
      ·                        ·                      ·
    · ● ·                    ·  ●                   ·   ·
  ·   ●   ·                ·    · ·               ·  ●  ·
    · ● ·                    ·  ·                    ·   ·
      ·                        ·                       ·

중심에서 균등 분산         회전하며 퍼짐            중력 무시, 떠다님
```

## Part 2: 12가지 마법 모션 원칙

디즈니 12원칙을 마법 세계에 맞게 재해석한 Pairy만의 원칙입니다.

### 원칙 1: 부드러운 시작과 끝 (Soft Ease)

**절대로 급격하게 시작하거나 끝나지 않습니다.**

```css
/* ❌ 잘못된 예: 선형 또는 급격한 변화 */
transition: all 0.3s linear;
transition: all 0.3s ease;

/* ✅ 올바른 예: 마법 이징 */
transition: all 0.3s cubic-bezier(0.25, 0.8, 0.25, 1);
```

```
속도 곡선 비교:

linear (기계적)     ease (일반적)      magic-soft (마법적)
   │ ────────        │    ___           │      ___
   │                 │  ╱   ╲           │   ╱     ╲
   │                 │╱       ╲         │ ╱         ╲___
   └──────────       └─────────         └─────────────────

냉정하고 기계적      평범함              따뜻하고 여운 있음
```

### 원칙 2: 다층 레이어 (Multi-Layer)

**하나의 효과는 여러 레이어로 구성됩니다.**

```
단일 레이어 (❌)              다층 레이어 (✅)

    [아이콘]                   Layer 4: 파티클 (0.6s)
                              Layer 3: 글로우 (0.4s)
                              Layer 2: 후광 (0.3s)
                              Layer 1: 아이콘 (0.2s)

단조로움                      깊이감과 풍부함
```

#### 레이어 구성 원칙
```
Layer 1 (Base):     아이콘/주요 요소의 상태 변화
Layer 2 (Ambient):  배경 글로우, 후광
Layer 3 (Effect):   주요 시각 효과 (빛, 파티클)
Layer 4 (Accent):   강조 파티클, 반짝임
Layer 5 (Trail):    잔상, 여운 효과
```

### 원칙 3: 빛 우선 (Light First)

**모든 상태 변화는 빛의 변화로 시작합니다.**

```
상태 전환 순서:

1단계: 빛이 먼저 반응
       (글로우 시작, 밝기 증가)

2단계: 형태가 반응
       (스케일, 위치 변화)

3단계: 파티클이 반응
       (요정 가루, 별 등 생성)

4단계: 여운이 남음
       (잔상, 페이드 아웃)
```

### 원칙 4: 유기적 타이밍 (Organic Timing)

**기계적으로 정확한 타이밍을 피합니다.**

```
기계적 타이밍 (❌)           유기적 타이밍 (✅)

파티클1: 0.0s               파티클1: 0.00s
파티클2: 0.1s               파티클2: 0.08s
파티클3: 0.2s               파티클3: 0.19s
파티클4: 0.3s               파티클4: 0.25s
파티클5: 0.4s               파티클5: 0.36s

정확히 0.1s 간격            약간씩 불규칙한 간격
로봇 같음                   살아있는 느낌
```

#### 스태거 딜레이 공식
```javascript
// 기계적 (❌)
delay = index * 0.1;

// 유기적 (✅)
delay = index * 0.08 + Math.random() * 0.04;
```

### 원칙 5: 호흡하는 대기 (Breathing Idle)

**정지 상태에서도 미세하게 살아있습니다.**

```css
/* 활성 상태의 아이콘은 숨쉬듯 움직임 */
@keyframes breathe {
  0%, 100% {
    transform: scale(1);
    filter: brightness(1);
  }
  50% {
    transform: scale(1.02);
    filter: brightness(1.05);
  }
}

animation: breathe 3s var(--glow-pulse) infinite;
```

### 원칙 6: 감정의 온도 (Emotional Temperature)

**모션은 감정을 전달합니다. Pairy의 감정은 따뜻합니다.**

| 감정 | 표현 방법 | 컬러 |
|------|----------|------|
| 사랑/좋아요 | 심장처럼 펄스, 따뜻한 글로우 | Pink |
| 수집/저장 | 안정적인 빛, 보호하는 느낌 | Violet |
| 기쁨 | 파티클 폭발, 반짝임 | Gold |
| 성공 | 후광 확산, 부드러운 빛 | White + Pink |

### 원칙 7: 자연의 리듬 (Natural Rhythm)

**자연에서 영감을 받은 움직임입니다.**

```
참고할 자연 현상:

반딧불이    - 불규칙한 깜빡임, 떠다니는 움직임
벚꽃잎      - 천천히 떨어지며 회전, 바람에 흔들림
오로라      - 부드러운 색 변화, 물결치는 형태
별빛        - 반짝임(twinkle), 명멸
물결        - 퍼져나가는 동심원, 점점 약해짐
안개        - 천천히 퍼지고 사라짐
```

### 원칙 8: 중심에서 확산 (Center Expansion)

**효과는 중심에서 시작하여 바깥으로 퍼집니다.**

```
확산 패턴:

시간 →

t=0     t=0.2    t=0.4    t=0.6    t=0.8

  ●       ◉        ◎        ○         ·

중심     확산      정점     약화      소멸
```

### 원칙 9: 잔상과 여운 (Afterglow)

**효과가 끝난 후에도 흔적이 남습니다.**

```css
/* 메인 효과 */
.main-effect {
  animation: burst 0.4s var(--fairy-dust) forwards;
}

/* 잔상 효과 - 메인보다 길게 */
.afterglow {
  animation: fade 0.8s var(--glow-fade) forwards;
  animation-delay: 0.3s; /* 메인 효과 중간부터 시작 */
}
```

### 원칙 10: 예측 가능한 마법 (Predictable Magic)

**마법이지만 사용자가 예측할 수 있어야 합니다.**

```
좋은 예측 가능성:

1. 호버 시 → 미리보기 (약한 글로우)
2. 클릭 시 → 본 효과 (강한 글로우 + 파티클)
3. 완료 시 → 안정 상태 (지속적인 약한 글로우)

사용자는 각 단계에서 무엇이 일어날지 직감적으로 앎
```

### 원칙 11: 절제된 화려함 (Restrained Brilliance)

**화려하지만 과하지 않습니다.**

```
파티클 수 가이드:

최소 효과:  3-5개 파티클
표준 효과:  6-10개 파티클
강조 효과:  10-12개 파티클
최대 제한:  15개 (절대 초과 금지)

글로우 강도:
호버:       opacity 0.3
활성:       opacity 0.5-0.7
강조:       opacity 0.8 (최대)
```

### 원칙 12: 일관된 마법 언어 (Consistent Magic Language)

**같은 의미는 같은 모션으로 표현합니다.**

| 의미 | 모션 언어 |
|------|----------|
| 활성화 | 중심에서 확산하는 글로우 |
| 비활성화 | 바깥에서 중심으로 수렴, 소멸 |
| 호버 | 미세한 스케일 업 + 약한 글로우 |
| 성공 | 파티클 버스트 + 후광 |
| 강조 | 펄스 애니메이션 |
| 로딩 | 회전하는 마법진 |

## Part 3: 이징 함수 심층 분석

### 3.1 마법 이징 라이브러리

```css
:root {
  /* ═══════════════════════════════════════════════════════
     기본 마법 이징 (Basic Magic Easing)
     ═══════════════════════════════════════════════════════ */

  /* 부드러운 마법 - 가장 기본적인 마법 움직임
     특징: 천천히 시작, 부드럽게 감속
     용도: 아이콘 상태 변화, 기본 트랜지션 */
  --magic-soft: cubic-bezier(0.25, 0.8, 0.25, 1);

  /* 떠오르는 효과 - 중력을 거스르는 움직임
     특징: 가볍게 시작, 우아하게 정착
     용도: 위로 떠오르는 요소, 등장 애니메이션 */
  --magic-float: cubic-bezier(0.4, 0, 0.2, 1);

  /* ═══════════════════════════════════════════════════════
     강조 이징 (Accent Easing)
     ═══════════════════════════════════════════════════════ */

  /* 반짝임 - 살짝 오버슈트하는 탄성
     특징: 목표를 살짝 넘었다가 돌아옴 (1.56 > 1.0)
     용도: 강조 효과, 주목을 끌어야 할 때 */
  --magic-sparkle: cubic-bezier(0.34, 1.56, 0.64, 1);

  /* 요정 날갯짓 - 더 큰 탄성
     특징: 더 강한 오버슈트 (1.275 > 1.0)
     용도: 버튼 활성화, 중요한 상태 변화 */
  --fairy-flutter: cubic-bezier(0.175, 0.885, 0.32, 1.275);

  /* ═══════════════════════════════════════════════════════
     확산 이징 (Expansion Easing)
     ═══════════════════════════════════════════════════════ */

  /* 요정 가루 - 빠르게 시작, 천천히 퍼짐
     특징: 초반 가속 후 길게 감속
     용도: 파티클 확산, 글로우 퍼짐 */
  --fairy-dust: cubic-bezier(0.16, 1, 0.3, 1);

  /* ═══════════════════════════════════════════════════════
     리듬 이징 (Rhythm Easing)
     ═══════════════════════════════════════════════════════ */

  /* 글로우 맥동 - 숨쉬는 듯한 리듬
     특징: 대칭적인 가속/감속
     용도: 반복 애니메이션, 호흡 효과 */
  --glow-pulse: cubic-bezier(0.4, 0, 0.6, 1);

  /* 글로우 페이드 - 자연스러운 소멸
     특징: 천천히 사라지는 여운
     용도: 페이드 아웃, 잔상 효과 */
  --glow-fade: cubic-bezier(0.25, 0.46, 0.45, 0.94);
}
```

### 3.2 이징 시각화

```
cubic-bezier 그래프 해석:

진행도
1.0 ┤                    ●────── magic-sparkle (오버슈트)
    │                 ●╱
    │              ●╱
0.5 ┤           ●╱
    │        ●╱
    │     ●╱─────────────────── magic-soft (부드러운 감속)
0.0 ┼●───────────────────────
    0.0              0.5              1.0  시간

--magic-sparkle: 1.0을 넘어갔다가 돌아옴 = 탄성 느낌
--magic-soft: 1.0에서 부드럽게 정착 = 안정적 느낌
```

### 3.3 이징 조합 패턴

```
다층 애니메이션 이징 조합:

Layer 1 (아이콘):     --magic-soft      (안정적)
Layer 2 (글로우):     --glow-fade       (부드러운 확산)
Layer 3 (파티클):     --fairy-dust      (빠른 시작, 느린 끝)
Layer 4 (반짝임):     --magic-sparkle   (탄성)

각 레이어가 다른 리듬을 가져 풍부한 움직임 생성
```

## Part 4: 타이밍 설계

### 4.1 기본 타이밍 단위

```
마법 시간 단위:

--timing-instant:   0.1s   (즉각 반응, 호버 등)
--timing-swift:     0.15s  (빠른 상태 변화)
--timing-normal:    0.2s   (기본 트랜지션)
--timing-smooth:    0.3s   (부드러운 효과)
--timing-gentle:    0.4s   (우아한 움직임)
--timing-slow:      0.6s   (강조 효과)
--timing-dramatic:  0.8s   (극적 효과)
--timing-ambient:   1.0s+  (배경 애니메이션)
```

### 4.2 상태별 타이밍 가이드

```
┌─────────────────────────────────────────────────────────┐
│                    클릭 이벤트 타임라인                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  0ms    100ms   200ms   300ms   400ms   500ms   600ms  │
│   │       │       │       │       │       │       │    │
│   ├───────┴───────┤                                    │
│   │  아이콘 변화   │ (0.15s, magic-soft)                │
│   │               │                                    │
│   ├───────────────┴───────┤                            │
│   │      스케일 펄스       │ (0.2s, fairy-flutter)      │
│   │                       │                            │
│   │   ├───────────────────┴───────────┤               │
│   │   │        글로우 확산             │ (0.3s)         │
│   │   │                               │               │
```

## Part 10: 금지 사항 (Anti-Patterns)

```
❌ 기계적 모션

├─ 뒤집기 (flip)
├─ 심한 흔들림 (shake)
├─ 튕기기 (bounce back)
└─ 급격한 스케일 변화 (0.5x 이하 또는 2x 이상)

❌ 브랜드 불일치
├─ 어두운 색상 글로우
├─ 네온/사이버펑크 스타일
├─ 차가운 색감 (파란색 과다)

```
