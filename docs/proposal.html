"페어를 완성하는 마법"

자캐러를 위한 틀 아카이브 + 페어틀 웹 에디터 + 실시간 협업 플랫폼

v2.0 - 피드백 반영 버전

📋 서비스 개요

한 줄 정의

"자캐러를 위한 틀 아카이브 + 페어틀 웹 에디터"

트위터/포스타입에 흩어진 이메레스, 트레틀, 페어틀을 한 곳에서 검색하고,
포토샵 없이 웹에서 바로 페어틀을 편집할 수 있는 플랫폼

타겟 유저

구분 대상 특성
핵심 타겟 트위터 자캐 커뮤러 모바일 중심, 페어 문화 활발
확장 타겟 TRPG 유저, 창작러 캐릭터 시트, 관계도 니즈

핵심 가치 (경쟁 우위)

기존 문제 페어리의 해결책
틀이 트위터/포스타입에 흩어져 있음 한 곳에서 통합 검색
포토샵 없으면 페어틀 못 씀 웹 에디터 제공
혼자 만들어야 함 실시간 협업 기능
무단 도용 걱정 워터마크 + 보호 서비스

✨ 핵심 기능 개선

🔍 1. 틀 아카이브

MVP 버전:

  • 카테고리: 이메레스, 트레틀, 페어틀, 프로필틀
  • MVP 집중: 2인 페어틀에 먼저 집중
  • 태그 시스템: #귀여움 #다크 #커플 #우정 등
  • 원작자 외부 링크 연결 (트위터/포스타입)
  • 좋아요 / 북마크 기능

확장 버전:

  • 틀 직접 업로드
  • 작가 프로필 페이지
  • 유료 틀 판매

📁 2. 틀 파일 3중 분리 시스템 NEW

문제: 외부 호스팅 → 다운로드 → 재업로드 과정에서 UX 단절

해결: 작가가 틀 등록 시 3가지 버전으로 분리 업로드

구분 형식 용도
① 미리보기용 JPG/PNG (압축) 검색 결과, 썸네일
② 에디터용 JSON/SVG (레이어 분리) 페어리 에디터에서 즉시 편집
③ 원본 소장용 PSD 외부 링크 구매자 다운로드

→ 결제 즉시 "에디터로 열기" 버튼 활성화, 플랫폼 이탈 방지

🎨 3. 페어틀 웹 에디터 개선

MVP 버전:

  • 템플릿 선택 (5~10개, 2인 페어틀 집중)
  • 이미지 드래그 & 드롭
  • 텍스트 추가/편집
  • 완성본 PNG 저장
  • NEW 원클릭 테마컬러 변경
  • NEW 자동 저장 (로컬스토리지 + DB)

확장 버전:

  • 레이어 편집
  • 필터/스티커
  • 자유 템플릿

🎨 원클릭 테마컬러 변경 기능 킬러 피처

Pain Point: 이메레스 사용 시 "내 캐릭터 테마컬러로 바꾸고 싶다"

해결:

  • 작가가 업로드 시 "변경 가능한 색상 영역" 지정
  • 사용자는 컬러 피커로 클릭 한 번에 테마색 변경
  • 배경색, 강조색, 텍스트색 등 영역별 커스터마이징

💾 자동 저장 시스템 NEW

  • 로컬스토리지: 5초마다 자동 저장 (비로그인도 가능)
  • DB 저장: 로그인 유저는 클라우드 동기화
  • 복구: 새로고침/재접속 시 "이전 작업 불러오기" 팝업
  • 버전 히스토리: 프리미엄 유저 최근 10개 버전 저장

📱 4. PC/모바일 이원화 전략 NEW

문제: 타겟 유저(트위터)의 대다수는 모바일 접속. 캔버스 에디터는 모바일에서 정교한 조작 어려움.

해결: 디바이스별 기능 분리

📱 모바일 (뷰어 + 간편 편집)

  • 틀 검색/탐색
  • 좋아요/북마크
  • 간편 편집 (텍스트, 이미지 넣기)
  • 협업 링크 공유
  • 완성작 저장/공유

"디테일한 편집은 PC 권장" 안내 표시

💻 PC (풀 에디터)

  • 전체 기능 지원
  • 정밀 크기 조절/회전
  • 레이어 순서 변경
  • 실시간 협업 (커서 표시)
  • 고화질 출력

👥 5. 실시간 협업 (핵심 차별화)

MVP 버전:

  • 링크 공유로 협업 초대
  • 동시 편집 (각자 영역)
  • 실시간 커서 표시
  • 완성 후 공동 저장

확장 버전:

  • 음성/화상 채팅
  • 버전 히스토리
  • 3인 이상 협업

🛡️ 6. 보호 서비스 개선

기존 문제: AI 노이즈(Glaze 스타일)는 GPU 필요, 비용 과다

현실적 대안: 가벼운 보호 기능으로 대체

  • AI 노이즈 (Glaze) → 패턴 오버레이 (미세 격자무늬)
  • 워터마크 타일링 (반투명 로고 반복)
  • 자동 워터마크 삽입
  • 우클릭 방지 옵션
  • 원작자 인증 뱃지
  • 신고/차단 시스템

🚀 유저 플로우

시나리오: "페어틀 같이 채우자!"

1️⃣ A가 페어리에서 마음에 드는 페어틀 발견 ↓ 2️⃣ "에디터로 열기" 클릭 (에디터용 JSON 즉시 로드) ↓ 3️⃣ 협업 링크 생성 → B에게 전송 ↓ 4️⃣ A는 왼쪽, B는 오른쪽 영역 편집 ↓ 5️⃣ 실시간으로 서로의 작업 확인 (자동 저장 중) ↓ 6️⃣ 완성! → PNG 저장 → 트위터에 공유
