## 1. CREPE (크레페)

### 1.1 서비스 개요

- **URL**: crepe.so (추정)
- **핵심 기능**: 커미션 중개, 슬롯 관리, 포인트 시스템
- **수익 모델**: 멤버십 구독 + 이용권 판매 + 포인트 충전

### 1.2 가격 정책

| 플랜 | 가격 | 주요 혜택 |
|------|------|----------|
| 그린 멤버십 | 1,900 P/월 | 슬롯 8개, 커미션 타입 8개, 배지 |
| 퍼플 멤버십 | 2,900 P/월 | 그린 + 추가 혜택 |
| 이용권 (1일) | 5,000~10,000 | 단기 이용 |
| 이용권 (7일) | 20,000~40,000 | 중기 이용 |
| 이용권 (30일) | 50,000~99,000 | 장기 이용 |

### 1.3 사용성 10원칙 평가

#### 상태 가시성 (Visibility of System Status)
| 항목 | CREPE | Pairy | 승자 |
|------|-------|-------|------|
| 현재 플랜 표시 | 프로필에 Lv.1 배지 상시 표시 | 구독 페이지에서만 확인 | **CREPE** |
| 사용량 피드백 | 슬롯 2개, 커미션 0/4 실시간 표시 | 내보내기 횟수 표시 | **CREPE** |
| 잔액/포인트 | 상단 0P 상시 표시 | 없음 | **CREPE** |

**인사이트**: CREPE는 레벨/포인트/슬롯을 **상시 노출**하여 사용자 현재 상태 파악 용이

#### 사용자 제어권 (User Control & Freedom)
| 항목 | CREPE | Pairy | 승자 |
|------|-------|-------|------|
| 구독 해지 | 별도 안내 없음 (불명확) | 버튼 1개로 명확 | **Pairy** |
| 작업 취소 | 커미션 특성상 제한적 | 에디터 Undo 지원 | **Pairy** |
| 뒤로가기 | 모달 중첩으로 혼란 | 자연스러운 네비게이션 | **Pairy** |

#### 일관성과 표준 (Consistency & Standards)
| 항목 | CREPE | Pairy | 승자 |
|------|-------|-------|------|
| 아이콘 일관성 | 다양한 스타일 혼재 | Lucide 아이콘 통일 | **Pairy** |
| 색상 체계 | 보라/초록/검정 혼합 | 딸기크림 파스텔 일관 | **Pairy** |
| 메뉴 구조 | 10개+ 메뉴 산발적 | 6개 탭 정리됨 | **Pairy** |

#### 미니멀리즘 디자인 (Aesthetic & Minimalist Design)
| 항목 | CREPE | Pairy | 승자 |
|------|-------|-------|------|
| 시각적 노이즈 | 높음 (이용권 카드 과다) | 낮음 (여백 활용) | **Pairy** |
| 핵심 CTA 집중 | 12개+ 상품 혼재 | "프리미엄 시작" 명확 | **Pairy** |
| 정보 밀도 | 과밀함 | 적절함 | **Pairy** |

**인사이트**: CREPE 스토어는 18개+ 상품 → **선택 과부하(Choice Overload)** 발생

### 1.4 사용자 여정 비교

#### 유료 전환 여정

**CREPE** (뎁스: 5+)
```
드롭다운 → 스토어 → 탭 탐색 → 멤버십 vs 이용권 이해 → 상품 선택(18개+) → 결제
```

**Pairy** (뎁스: 3)
```
프리미엄 메뉴 → 플랜 선택(3개) → 결제
```

| 마찰 요인 | CREPE | Pairy |
|----------|-------|-------|
| 개념 이해 | 포인트/마일리지/수익금 혼란 | 단일 구독 모델 |
| 선택지 | 1일/7일/30일 × 일반/성인 = 12개 | 2개 플랜 |
| 가격 표기 | "P" (포인트? 원화?) | ₩ 원화 명확 |

#### 크리에이터 수익 확인 여정

**CREPE** (뎁스: 3)
```
프로필 → 드롭다운 → 수익금 → 포인트/수익금/마일리지 구분 필요
```

**Pairy** (뎁스: 2)
```
마이페이지 → 크리에이터 탭 (대시보드 통합)
```

### 1.5 UX 라이팅 비교

| 상황 | CREPE | Pairy | 분석 |
|------|-------|-------|------|
| CTA | "구독하기" | "프리미엄 시작" | Pairy 더 능동적 |
| 빈 상태 | "아직 커미션 타입이 없어요." | "아직 작업이 없어요" | 동등 |
| 유도 | "새로운 커미션을 만들어 보세요." | "더 멋진 작품을 만들어보세요." | Pairy 감성적 |
| 가격 | "1,900 P" | "₩2,900/월" | Pairy 명확 |

#### 톤앤매너
| 특성 | CREPE | Pairy |
|------|-------|-------|
| 말투 | 격식체 (~합니다) | 친근체 (~해요) |
| 감성 | 중립적/사무적 | 따뜻함/귀여움 |
| 이모지 | 거의 없음 | 적절히 활용 |

### 1.6 정보 구조(IA) 비교

**CREPE IA** (복잡)
```
헤더: 검색 / 셔플 / 메일 / 프로필
  └─ 드롭다운 (10개 메뉴)
     ├─ 내 프로필
     ├─ 북마크 및 오픈 알림
     ├─ 캘린더
     ├─ 포인트
     ├─ 수익금
     ├─ 스토어
     ├─ 신청자 메뉴
     ├─ 커미션주 메뉴
     ├─ 설정
     └─ 로그아웃
```

**Pairy IA** (단순)
```
헤더: 로고 / 틀 둘러보기 / 내 작업 / 프리미엄 / 알림 / 프로필
  └─ 마이페이지 탭 (6개)
     ├─ 프로필
     ├─ 내 작업
     ├─ 북마크
     ├─ 구독
     ├─ 크리에이터
     └─ 설정
```

### 1.7 CREPE 강점 (벤치마킹 포인트)

| 기능 | 설명 | Pairy 적용 방안 |
|------|------|----------------|
| 레벨 배지 | 프로필에 Lv.1 상시 표시 | 헤더 아바타 옆에 구독 티어 아이콘 |
| 슬롯 표시 | "현재 남은 슬롯 2개" 실시간 | 내보내기 잔여 횟수 위젯 |
| 도전 배지 | 활동 기반 성취 시스템 | 배지 시스템 도입 검토 |
| 캘린더 | 일정/작업 관리 | 작업 히스토리 캘린더 뷰 |
| 포인트 시스템 | 다양한 결제 수단 | 현재는 단순 구독 유지 |

### 1.8 CREPE 약점 (Pairy 차별화 포인트)

| 약점 | 설명 | Pairy 강점 |
|------|------|-----------|
| 복잡한 가격 체계 | 멤버십+이용권+포인트 혼재 | 단순 월정액 2개 |
| 개념 과부하 | P/마일리지/수익금 구분 필요 | 원화 통일 |
| 시각적 혼잡 | 스토어에 18개+ 상품 | 3개 플랜 비교 |
| 학습 비용 | 10개+ 메뉴 이해 필요 | 직관적 6개 탭 |
